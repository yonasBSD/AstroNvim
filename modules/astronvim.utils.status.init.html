Module astronvim.utils.status.init

AstroNvim Status Initializers

Statusline related init functions for building dynamic statusline components

This module can be loaded with local init = require "astronvim.utils.status.init"

Functions

breadcrumbs(opts)
An init function to build a set of children components for LSP breadcrumbs

Parameters:

  • opts ? table # options for configuring the breadcrumbs (default: { max_depth = 5, separator = "  ", icon = { enabled = true, hl = false }, padding = { left = 0, right = 0 } })

Returns:

    function # The Heirline init function

Usage:

    local heirline_component = { init = require("astronvim.utils.status").init.breadcrumbs { padding = { left = 1 } } }
separated_path(opts)
An init function to build a set of children components for a separated path to file

Parameters:

  • opts ? table options for configuring the breadcrumbs (default: { max_depth = 3, path_func = provider.unique_path(), separator = "  ", suffix = true, padding = { left = 0, right = 0 } })

Usage:

    local heirline_component = { init = require("astronvim.utils.status").init.separated_path { padding = { left = 1 } } }
update_events(opts)
An init function to build multiple update events which is not supported yet by Heirline's update field

Parameters:

  • opts any[] an array like table of autocmd events as either just a string or a table with custom patterns and callbacks.

Returns:

    function # The Heirline init function

Usage:

    local heirline_component = { init = require("astronvim.utils.status").init.update_events { "BufEnter", { "User", pattern = "LspProgressUpdate" } } }
